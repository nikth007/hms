How the people columns work — this is the project's "who does what" in plain words:
  • Doing it — the owner: the person (or pair) actually doing the work. Every task needs one.
  • Signs off — the one person who confirms it is properly done. Keep this to a single name — if two people sign off, neither does.
  • Needs to be asked — people whose input is needed before or while doing it (e.g. JMT for TAT decisions). Ask them early; don't surprise them at sign-off.
  • Gets update — people simply kept in the loop once things move. They don't need to act.
Colours follow a traffic light: red = overdue or stuck, amber = due within 3 days or missing a date, green = on track or done. Click any cell to edit; click the due chip to pick a date from the calendar.

Live sync — updates on every device, instantly

With live sync ON, any change anyone makes appears on everyone else's open tracker within a couple of seconds. Only one person needs to do this setup, once. Everyone else just opens the share link.

  1. Go to console.firebase.google.com and sign in with any Google account → Create a project (free, no card needed). Skip Analytics.
  2. In the left menu: Build → Realtime Database → Create database → pick a location → choose "Start in test mode".
  3. Copy the database URL shown at the top (looks like https://yourproject-default-rtdb.firebaseio.com/) and paste it below.
  4. Click Connect, then use "Copy share link" in the top bar — that link carries the connection, so colleagues who open it join live sync automatically.

Note: test mode expires after 30 days — in Firebase, open the Rules tab and set both .read and .write to true to keep it running. Anyone who has the link can edit, so share it only within the team and don't store anything confidential.
